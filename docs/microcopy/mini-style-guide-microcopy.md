# Mini Style Guide de Microcopy — Webapp Bússola

## 1) Voz e tom
- **Institucional:** alinhado ao protocolo escolar oficial.
- **Acolhedor:** reconhece urgência sem gerar pânico.
- **Direto:** frases curtas, ação clara.
- **Respeitoso:** orienta sem culpabilizar profissionais, estudantes ou famílias.

## 2) Princípios operacionais
1. **Clareza imediata:** quem lê precisa entender em até 3 segundos.
2. **Ação explícita:** botão e instrução devem dizer o que acontece depois.
3. **Consistência terminológica:** mesmo conceito, mesmo termo.
4. **Redução de ansiedade:** alertar risco sem linguagem alarmista excessiva.
5. **Acessibilidade textual:** evitar referência visual isolada (ex.: “clique no botão verde”).

## 3) Estrutura de frases
- Limite recomendado: **até 20 palavras** por frase.
- Preferir **voz ativa**.
- Um conceito por frase.
- Para sequência crítica: usar lista numerada (1, 2, 3).

## 4) Padrão para botões e CTAs
- Evitar: `OK`, `Confirmar`, `Salvar`, `Cancelar` sem contexto.
- Preferir: verbo + objeto + consequência curta.

### Exemplos
- `Confirmar` → **`Abrir protocolo oficial`**
- `Cancelar` (em modal crítico) → **`Voltar ao resultado`**
- `Entendi / Concluir` → **`Concluir e reiniciar triagem`**
- `Cancelar` (modal de emergência) → **`Fechar sem ligar`**

## 5) Mensagens de erro (template)
**[O que aconteceu] + [causa provável] + [como corrigir]**

Exemplo:
- “Não foi possível salvar o registro. Verifique sua conexão e tente novamente.”

## 6) Terminologia oficial padronizada
| Conceito | Usar | Evitar |
|---|---|---|
| Órgão de proteção | Conselho Tutelar (primeira ocorrência) / CT (depois) | conselho, conselheiro |
| Saúde mental infantojuvenil | CAPS Infantojuvenil (CAPS IJ) | CAPS criança |
| Registro formal | ocorrência / registro | caso, ticket |
| Acionamento urgente | notificar / acionar | chamar, avisar |
| Pessoa atendida | estudante / aluno(a) | cliente, usuário |

## 7) Templates reutilizáveis
- **Confirmação irreversível:**
  - “Ao confirmar, [ação externa] será iniciada hoje. Deseja continuar?”
- **Checklist de encerramento:**
  - “Antes de concluir, confirme: [ação 1], [ação 2], [ação 3].”
- **Ação crítica com saída segura:**
  - Botão principal: “Executar [ação]”
  - Botão secundário: “Voltar ao resultado”
